Workflows

A workflow is an ordered series of steps that run on a Sprig durable runtime. Workflows can be triggered by HTTP, cron, queue messages, or directly from another workflow.

Create a workflow

Returns a Workflow resource. Idempotent on name.

POST /v1/workflows
Create a new workflow definition. The runtime begins accepting triggers immediately.
// Request
POST /v1/workflows
Authorization: Bearer sk_live_***
Content-Type: application/json

{
  "name": "order-fulfillment",
  "trigger": { "type": "http" },
  "steps": [
    { "name": "reserve-inventory", "retries": 3 },
    { "name": "charge-card" },
    { "name": "send-receipt" }
  ]
}

List workflows

Paginated. Returns up to 100 workflows per page.

GET /v1/workflows?cursor=&limit=50
Lists workflows owned by the authenticated workspace.
Indexing for AI agents

The OpenAPI feed lives behind a STILE medium-tier gate.

Sprig publishes a fully-typed OpenAPI 3.1 spec at /openapi.json. To prevent low-effort scraping while keeping the feed friendly to model-driven indexers, the endpoint is gated at tier=medium — agents must echo the embedded challenge word back to the verify endpoint.

GET /openapi.json
→ 401 stile-required
   X-STILE-Verify: /__stile-verify?token=…&word=quartz-ladder

GET /__stile-verify?token=…&word=quartz-ladder
→ 200 Set-Cookie: stile=…

GET /openapi.json
→ 200 (full spec, 1.2MB)
